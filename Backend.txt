# Final Backend Architecture (Supabase-Based)

---

# 1️⃣ Overall System Architecture

```
Flutter Mobile App
        │
        │
Voice Booking Layer
(speech_to_text + flutter_tts)
        │
        ▼
 Supabase Client SDK
        │
 ┌───────────────┬───────────────┬───────────────┐
 │               │               │               │
Auth           Database        Storage        Realtime
Service        (PostgreSQL)    Service        Updates
 │               │               │               │
 │               │               │               │
Demo Aadhaar   Equipment       Images        Live Tracking
Verification   Bookings        Damage        Notifications
               Payments        Reports
```

Supabase acts as:

* Authentication server
* PostgreSQL database
* API layer
* File storage
* Realtime update system

So **no separate backend server is required for the demo project**.

# 3️⃣ Authentication & Demo Verification

Authentication is handled by Supabase Auth.

User registration fields:

```
name
email
password
aadhaar_number
role
verification_status
created_at
```

Demo Aadhaar verification logic:

```
if length(aadhaar) == 12
   verification_status = VERIFIED_DEMO
else
   verification_status = INVALID
```

Authentication flow:

```
User Register
      ↓
Supabase Auth creates user
      ↓
User profile stored in database
      ↓
Demo Aadhaar validation
      ↓
User login enabled
```

Security:

* Supabase JWT tokens
* Row Level Security policies
* Password encryption handled by Supabase

# 7️⃣ Payment Service (Stripe)

Payment is handled through:

Stripe sandbox.

Payment flow:

```
Booking Accepted
      ↓
Flutter requests payment intent
      ↓
Supabase Edge Function
      ↓
Stripe Payment Intent created
      ↓
User pays
      ↓
Stripe Webhook
      ↓
Payment status updated
      ↓
Booking confirmed
```

Edge functions used for:

```
create-payment-intent
stripe-webhook-handler
```
